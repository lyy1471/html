✨ 教程概览

本教程将手把手教你如何使用 GitHub Pages 免费部署一个静态网页，只需几个 HTML 文件，无需购买服务器或备案，几分钟上线你的个人网页或项目站点。

适合对象

  • 不懂后端的前端初学者
  • 想要部署个人主页、作品集、项目文档的人
  • 对 Git 和 GitHub 有基本了解者

📁 一、准备你的网页文件

你需要准备一份包含如下文件的静态网页项目：

my-site/
├── index.html        # 首页，必须存在
├── style.css         # 可选
└── script.js         # 可选

💡 index.html 是网页的入口文件，GitHub Pages 默认会访问它。

🔧 二、创建 GitHub 仓库并上传网页文件

  1. 登录 GitHub，点击右上角 + → New repository

  2. 设置仓库信息：

    • Repository name: 如 my-site
    • Visibility: Public
    • 不勾选 Initialize

  3. 打开终端，上传你的文件：

cd my-site                   # 进入网页目录
git init                    # 初始化 Git 仓库
git remote add origin https://github.com/你的用户名/my-site.git
git add .
git commit -m "initial commit"
git branch -M main
git push -u origin main

🌐 三、启用 GitHub Pages 功能

  1. 打开你的仓库页面 → Settings → 左侧点击 Pages

  2. 找到 Source ，设置为：

    • Branch: main
    • Folder: / (root)

  3. 点击 Save，等少2秒系统创建网站，显示：

Your site is live at https://你的用户名.github.io/my-site/

🎉 点击链接，你的网页已经上线啦！

↺ 四、更新网页内容

以后更新只需修改文件，提交并 push：

git add .
git commit -m "update something"
git push

等少许秒，GitHub Pages 会自动重新部署。

🌍 五、绑定自定义域名 (Optional)

  1. 准备一个你拥有的域名

  2. 在仓库根目录新建文件 CNAME，内容是你的域名：

www.example.com
  1. 到 DNS 服务商设置 CNAME 记录：
类型主机记录值
CNAMEwww你的用户名.github.io
  1. 回到 GitHub Pages 看到绿锁即表示成功启用 HTTPS

🛠️ 常见问题排查

问题解决方法
页面 404检查 index.html 是否存在，等待创建
CSS 不生效相对路径是否正确
React/Vue 等前端框架需先构建后部署到 dist/
域名不生效检查 DNS 配置，加 CNAME 文件
